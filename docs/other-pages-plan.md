# 不现实商店其他页面规划

## 1. 先定总原则

首页已经负责三件事：

- 先让用户记住产品
- 再让用户感受到品牌世界
- 最后让用户建立信任

所以其他页面不要再重复首页，而是各自承担更明确的任务。

我建议后续页面按这个逻辑分工：

- `品类页 / Category Pages`：负责找商品
- `系列页 / Collection Pages`：负责理解世界观
- `商品详情页 / Product Page`：负责成交
- `About / 关于`：负责建立品牌认同
- `Shipping & FAQ / 配送与常见问题`：负责减少顾虑
- `Search / 搜索`、`Cart / 购物车`、`Account / 账号`：负责完成购买路径

## 2. 推荐站点结构

### 一级可见页面

- `Home / 首页`
- `Keycaps / 键帽`
- `Keyboards / 键盘`
- `Desk Mats / 桌垫`
- `Cups / 杯子`
- `Cases / 壳类`
- `About / 关于`

### 二级或功能页面

- `Collections / 系列总览`
- `Collection Detail / 单个系列页`
- `Product Detail / 商品详情页`
- `Shipping & FAQ / 配送与常见问题`
- `Search / 搜索`
- `Cart / 购物车`
- `Account / 账号`

### 后续可加页面

- `Community / 社区内容`
- `Archive / 过往档案`
- `Journal / 品牌日志`

专业判断：
`Community / 社区内容` 和 `Journal / 品牌日志` 不建议现在就做成重点页面。  
现在更值钱的是先把“找商品”和“下单前的信息完整度”做扎实。

## 3. 品类页规划

这个页面不是首页商品区的放大版，而是用户真正开始筛选和浏览的地方。

建议每个一级品类都用同一套模板：

- `Keycaps / 键帽`
- `Keyboards / 键盘`
- `Desk Mats / 桌垫`
- `Cups / 杯子`
- `Cases / 壳类`

### 页面结构

1. `Category Hero / 品类头图`
- 品类标题
- 一句短说明
- 一张代表性图

2. `Filter Bar / 筛选条`
- `Availability / 状态`
- `Collection / 系列`
- `Price / 价格`
- `Sort / 排序`

3. `Product Grid / 商品网格`
- 方图优先
- 统一卡片信息
- 显示价格、状态、系列

4. `Editorial Insert / 编辑型插入位`
- 每 8 到 12 个商品之间插入一张系列横幅或一句品牌文案

5. `Pagination or Load More / 翻页或加载更多`

### 品类页重点

`Keycaps / 键帽` 会是最重的页面，因为你的 SKU 最多。  
所以它需要额外支持：

- `Magic / 魔法`
- `Literary / 大文豪`
- `Entry / 世界入口`
- `In Stock / 现货`
- `Pre-order / 预售`

专业判断：
不要给每个品类页都做完全不同的结构。  
统一模板更像成熟商城，也更方便后续扩 SKU。

## 4. 系列总览页规划

这个页面负责告诉用户：
你卖的不只是商品，而是一组一组完整的世界。

页面名建议：

- `Collections / 系列`

### 页面结构

1. `Page Intro / 页面导语`
- 短标题
- 短说明

2. `Collection Grid / 系列网格`
- 每个系列一张卡
- 显示系列图、系列名、短说明、商品数量

3. `Featured Collection Strip / 主推系列横条`
- 放 1 到 2 个当前最值得推的系列

### 首批建议展示的系列

- `Magic Series / 魔法系列`
- `Literary Series / 大文豪系列`
- `Great Writer Keyboard / 大文豪键盘`
- `Desktop Extensions / 桌面延展`
- `World Entry / 世界入口`
- `Cup World / 杯欢世界`

## 5. 单个系列页规划

这是最关键的页面之一。  
它比品类页更有情绪，比商品详情页更有叙事。

### 页面结构

1. `Collection Hero / 系列首屏`
- 系列名
- 一句核心定义
- 一张代表图

2. `Collection Statement / 系列说明`
- 2 到 4 行短文
- 讲这个系列为什么存在

3. `Products in This Collection / 本系列商品`
- 商品网格
- 可按状态筛选

4. `Mood or Detail Strip / 气质或细节条`
- 局部图
- 包装图
- 工艺图

5. `Related Worlds / 相关系列`
- 引导用户继续看别的系列

### 适合先做的系列页

1. `Magic Series / 魔法系列`
2. `Literary Series / 大文豪系列`
3. `Great Writer Keyboard / 大文豪键盘`

专业判断：
系列页一定要比品类页更“编辑化”，但不能复杂到影响买东西。  
它应该像“带购买能力的品牌展页”。

## 6. 商品详情页规划

这是成交页，所以信息必须最完整。

### 页面结构

1. `Gallery / 图片区`
- 主图
- 细节图
- 包装图
- 使用图

2. `Buy Box / 购买区`
- 商品名
- 一句话短说明
- 价格
- 状态
- 购买按钮
- 支付和配送提示

3. `Variant Area / 规格区`
- 颜色
- 套装版本
- 键数或适配信息

4. `Product Story / 商品说明`
- 讲灵感来源
- 保持简洁

5. `Specs / 参数`
- 材质
- 尺寸
- 适配
- 包含内容

6. `Shipping Note / 配送提示`
- 发货地
- 预计时效
- 是否预售

7. `Related Products / 相关商品`

### 详情页必须显式出现的信息

- `In Stock / 现货`
- `Pre-order / 预售`
- `Coming Soon / 即将发售`
- `Sold Out / 售罄`

### 键盘类详情页额外需要

- 布局图
- 套件说明
- 轴体或兼容说明
- 键盘保护膜相关说明

## 7. About 页面规划

这个页面不是写品牌鸡汤，而是让第一次认识你的人理解：
为什么这个品牌不是普通店铺。

### 页面结构

1. `Brand Statement / 品牌定义`

2. `How We Work / 我们怎么做`
- 主题
- 命名
- 图像
- 工艺

3. `What Makes Oddity Store / 不现实商店的不同`

4. `Studio Channels / 工作室渠道`
- Instagram
- YouTube
- TikTok
- 其他账号

5. `Contact / 联系方式`

### 文案方向

保持短、硬、明确。  
不要写成长篇自述。

## 8. Shipping & FAQ 页面规划

这是海外站最重要的信息页之一。

### 页面结构

1. `Shipping Overview / 配送概览`
- 支持哪些地区
- 大致时效

2. `Order Types / 订单类型`
- 现货
- 预售
- 限定发售

3. `Customs & Taxes / 税费说明`

4. `Returns / 售后与退换`

5. `FAQ / 常见问题`
- 什么时候发货
- 怎么查询订单
- 是否支持合单
- 售罄是否补货

专业判断：
这个页面越清楚，海外转化越高。  
很多品牌站视觉很强，但就是死在这页不清楚。

## 9. Search / 搜索 规划

搜索不是可有可无，它会是 SKU 增加后的重要入口。

### 建议做法

- 点击顶部搜索图标后出现 `Search Overlay / 搜索浮层`
- 可以直接搜商品名、系列名、关键词
- 下面实时显示：
  - 商品
  - 系列
  - 最近浏览或推荐

### 搜索结果优先级

1. 商品
2. 系列
3. 内容页

## 10. Cart / 购物车 与 Account / 账号 规划

这两页在 V1 不需要做得很花。

### `Cart / 购物车`

重点信息：

- 商品缩略图
- 价格
- 数量
- 状态提示
- 预计配送提示

### `Account / 账号`

重点信息：

- 登录
- 注册
- 订单记录
- 地址

专业判断：
这两页早期可以偏简洁，把精力放在商品展示和详情信息上更划算。

## 11. V1 最值得先做的页面顺序

我建议按这个顺序推进：

1. `Category Page Template / 品类页模板`
2. `Product Detail Template / 商品详情模板`
3. `Collection Detail Template / 系列详情模板`
4. `Collections / 系列总览`
5. `Shipping & FAQ / 配送与常见问题`
6. `About / 关于`

原因很直接：

- 先有品类页，用户才能真正逛
- 先有详情页，网站才具备成交能力
- 系列页负责把品牌世界讲完整
- `Shipping & FAQ / 配送与常见问题` 解决海外用户顾虑

## 12. 我建议你下一步先做哪一个

如果现在继续推进，我最推荐下一步先做：

`Keycaps / 键帽品类页`

因为它最能代表你的网站未来会不会好用。  
只要这个模板定对了，后面的：

- `Keyboards / 键盘`
- `Desk Mats / 桌垫`
- `Cups / 杯子`
- `Cases / 壳类`

都能快速套进去。
