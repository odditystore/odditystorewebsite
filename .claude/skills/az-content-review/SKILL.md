# AZ内容评审系统

## 什么时候用这个skill

以下任一情况**必须**用这个skill做评审：

- 任何在 `docs/drafts/` 下写/改的AZ口播脚本、小红书图文、视频文案
- 讨论AZ号（AZ的不现实商店）或品牌号（不现实商店）的内容策略
- 用户让"帮我规划这条内容""帮我修改""哪种更好""挑毛病"
- 用户写了脚本初稿想让我看
- 讨论封面/标题/钩子
- 讨论发布时机、平台适配、系列规划

## 核心原则（最重要，不能忘）

### 1. 避免"成型陷阱"
每次评审开始前**先承认自己的盲点**。稿子越精致，越容易丧失批判神经。用户每次让"再挑毛病"，你要保持同样的锋利，不能因为"前面已经改过"就放水。

### 2. 战略层 > 执行层
**永远先问"该不该发"，再问"怎么改"**。执行优化（钩子/标题/结构）意义不大，如果战略错位（账号标签偏移/时机错配/身份冲突）。我在第一轮评审时就因为跳过战略层、直接做执行优化，漏掉了"这条可能把AZ号带偏到广告评论者身份"这个最关键的问题。

### 3. 按严重度排序，不堆列表
🔴严重→🟡中等→🟠轻量。每一层只讲最锋利的几个。堆10个小毛病淹没3个大毛病是劣质评审。

### 4. 每个问题必须给具体改进
不只说"钩子太弱"，要给重写的那句话。不只说"缺自嘲"，要给具体的自嘲台词。抽象诊断没有价值。

### 5. 事实必须基于项目记忆或AZ本人确认
`docs/az-content-strategy.md` 是AZ的完整项目记忆。所有出现在脚本里的AZ个人信息（经历/数据/公司名）必须在这个文件里能找到出处，或者标注 `[ ] 待AZ确认` 让他补。**不能自己编AZ的"可能这样"的故事**。

## 评审流程（8-pass）

按这个顺序过一遍。前面三pass是战略级，漏掉等于白做。

### Pass 1 · 战略层：这条该不该做

在所有细节之前问这三个：

1. **账号标签风险** — 这条如果爆了，算法会给AZ贴什么标签？跟他"主理人/造物者"的核心身份一致吗？
   - 例：讲广告行业会被贴"广告评论者"标签，偏离主理人
   - 例：讲AI工作流可以从"主理人用AI提效"角度切，标签不偏
   
2. **时机是否对** — AZ号当前粉丝量级适合这条吗？
   - 156粉起号期 → 高频+垂直+短平快，深度立场片ROI低
   - 1k-3k腰部期 → 立场片开始有效
   - 1万+ → 立场片能立行业身份
   
3. **长期人脉账** — 这条会关掉哪些未来的门？
   - 点名前东家 → 关掉回去的路，关掉部分合作
   - 批评某个圈子 → 关掉被该圈子接纳的可能

如果这一层判断"不该发"或"现在不该发"，**立刻告诉用户**。不要为了完成任务硬写。

### Pass 2 · 身份层：这条跟AZ的身份有什么关系

AZ的双号定位：

| | AZ的不现实商店 | 不现实商店 |
|---|---|---|
| 身份 | 主理人（人） | 品牌（货） |
| 调性 | 松弛、真实、自嘲+中二浪漫 | 认真中二、浪漫、仪式感 |
| 内容 | 踩坑过程方法日常 | 产品世界观视觉 |

**每条内容都必须回答：这条为什么是AZ讲，不是别人？** 如果答案是"谁都能讲"，这条不成立。AZ独有的弹药是：前爱德曼+前蓝标**群创意总监**/十几年广告PR背景/裸辞做键帽品牌/一个人从AI草图到量产。没有用到这些弹药的稿子，信用低一档。

### Pass 3 · AZ三必问

用户自己定的，不能跳：

1. **别人看完能学到什么？** 学到的是"观点"还是"干货"？立场片允许观点，但至少要有一个可学的思考框架
2. **别人看完能马上做什么？** 必须有一个**具体可执行的动作**。"你该XX"不算，"找个晚上录60秒讲XX"算
3. **这条跟AZ的身份有什么关系？** 见Pass 2

有任何一问答不上来的，脚本不合格。

### Pass 4 · 内容结构

AZ自定的硬结构：

- **前3秒** — 结果前置+钩子。先砸成绩/金句/反差/悬念。**不要"最近XX发了篇文章"这种交代背景的开头**
- **前30%** — 从观众痛点切入。"你是不是也这样"，不是"我怎样"。必须前5-10秒让目标观众认领"这是给我的"
- **中间50%** — 实操干货（干货片）或 具体画面/故事/例证（立场片）。**抽象判断必须配一个具体画面或可验证例证**，否则是口号
- **后20%** — 观点收束 + 互动引导 + 回身份锚

**检查项**：
- [ ] 第1句话就是钩子，不是铺垫
- [ ] 前10秒内明确目标观众
- [ ] 抽象结论后面都跟着具体画面
- [ ] 结尾有可做的动作 + 评论区钩 + 主理人身份回锚

### Pass 5 · 语气硬性要求

AZ定的调性规则，违反就不是AZ：

- [ ] 松弛像朋友喝酒，**不像讲座**
- [ ] 多"我当时""我以前"，少"你应该""你们"
- [ ] 至少一处明显的自嘲/自黑（讲自己蠢事）
- [ ] 别替观众总结道理，用画面让他们自己悟
- [ ] "我也不知道对不对 反正我是这么干的" > "更聪明的方式是"
- [ ] 不用"千万不要""你知道吗"这种模板句式

**常见违规**：
- 把金句当结论连发（像讲课）
- 扎心三连用"你们"作主语（变成指手画脚）
- 全篇严肃没有一处让观众笑的瞬间

### Pass 6 · 信任破绽检查

立场片最怕的不是被反驳，是被**暗暗怀疑但没说出来**。找出观众心里可能有的那句"那你自己呢？"主动处理。

**常见破绽**：
- AZ号目前156粉——讲"怎么做内容"会被想"你自己也没做起来"
- 讲"快速测试"——会被想"你不就是靠AI省事吗"
- 讲"离职一年半"——会被想"你是不是没工作了在装"

**处理原则**：**主动自揭 > 回避 > 强辩**。诚实是最强的修辞。一句"我这号现在156粉 也没做起来 但156粉总比零条强"的力量 > 绕着不提。

### Pass 7 · 事实准确性

任何写进脚本的AZ个人信息必须对照 `docs/az-content-strategy.md`：

- ✅ 爱德曼国际公关 群创意总监
- ✅ 蓝色光标 群创意总监
- ✅ 干了十几年
- ✅ 管过30号人团队
- ✅ 第一款键帽启示魔法日报3天卖200套
- ✅ 不现实商店品牌13万粉
- ✅ AZ号当前粉丝数（查项目记忆实时数据）

**不允许**：
- 自己编AZ"可能经历过"的场景
- 把品牌号数据说成AZ个人号数据（之前犯过："我一个人13万"——这是品牌号，不是AZ号）
- 把公司冤枉成反例（之前犯过：把TBWA自嘲当扎心武器，但那句话本身就是TBWA做对的事）
- 不经AZ确认披露新的个人信息（竞业/股权/关系）

### Pass 8 · 留存曲线预判

把脚本切成5-15秒一段，给每段打留存分（想象小红书观众）：

```
0-5秒：   钩子硬度 _/10
5-15秒：  观众认领 _/10
15-30秒： 信息密度 _/10
30-60秒： 最强钩（故事/反转） _/10  ← 关键段
60-90秒： 判断收束 _/10
90-115秒：落点+互动 _/10
```

**危险信号**：
- 30-60秒如果没有最强钩（画面/反转/情绪高点），整条片留存崩
- 超过20秒没有任何钩（画面/金句/反差），观众划走

## 平台载体差异（每个出口的正文都要分开写）

**核心原则**：**脚本 ≠ 笔记正文 ≠ 标题**。三个东西三种写法。

| 载体 | 长度 | 作用 | 写法 |
|---|---|---|---|
| 视频脚本（口播） | 按平台1-2分钟 | 讲完整的事 | 口语化、有节奏、有情绪 |
| 小红书笔记正文 | **3-7行** | 钩子+关键词+互动引导 | **不复述脚本**，挑出最扎的一句+悬念+评论区钩 |
| 抖音/视频号描述 | 1-2行+话题 | 定位+关键词 | 超短 |
| B站简介 | 2-4行 | 补充视频没讲完的 | 可以略长 |
| 封面标题 | 4-10字 | 停留钩子 | 极简、有悬念或反差 |
| 笔记标题 | 15-20字 | 搜索+点击率 | 带关键词+情绪词 |

### 小红书笔记正文的硬规则

❌ **不要**：把脚本复制成文字版（太长 / 看过视频的人再看是冗余 / 没看视频的人没有想看的钩子）

✅ **要**：
- **3-7行**，留白
- **第1行是钩子**（最扎或最有悬念的一句）
- **中间1-2句埋核心观点**（让没看视频的人也get到一半）
- **留一个悬念**（完整答案在视频里）
- **最后一行是互动钩**（评论区提问）
- **关键词藏在自然句子里**（给算法抓但不生硬）

### 示范对比

❌ 差的正文（脚本复制版）：
```
最近数英发了篇文章叫《4A公司怎么做小红书 快来抄作业》
我点进去的时候愣了一下
这张图里的六家公司 有一家是我以前的东家
我在爱德曼国际公关做过群创意总监
在蓝色光标也做过群创意总监
所以今天讲的这件事 也是在讲我自己
[...后面600字]
```

✅ 好的正文（钩子版）：
```
前蓝标群创意总监
这张图里有一家是我前东家

我干了十几年广告 才想明白一件事——
我们不是不会做 是肌肉长反了

你干广告这几年 最气的是哪次
评论区告诉我 我在做下一款键帽
```

### 评审时必查

写完脚本后，**必须单独写一遍小红书笔记正文**——不能只给脚本。每条存稿必须包含：
- [ ] 完整脚本
- [ ] 独立的小红书笔记正文（3-7行 钩子版）
- [ ] 封面标题（4-10字）
- [ ] 笔记标题（15-20字）
- [ ] 标签（10-15个 含关键词+情绪词+身份标签）

## 常见问题模式（我在对话里反复发现的）

### 🔴 模式1：开场交代背景而非钩子

```
❌ "最近XX发了篇文章叫《…》我点进去看了一下"
✅ "图里有我。" / "六家加起来不到5万粉。"
```

### 🔴 模式2：动作不安全

给观众的action是否会让他们真的踩坑？
```
❌ "录一个被毙的idea" ← 在职的人不敢，涉及保密
✅ "讲你最气的一次" ← 情绪型，零风险
```

### 🔴 模式3：结论型口号没有具体例证

```
❌ "只学了壳没学到里子"
✅ "标题还是'3招教你''一文读懂'——这是公众号句式不是小红书的"
```

### 🔴 模式4："我 vs 你们"的指手画脚

```
❌ "以前你们定义什么叫好内容 现在你们做不过甲方实习生"
✅ "我当年以为拿个戛纳就够了 后来发现小红书那个博主一条吐槽比我戛纳作品火"
```

### 🔴 模式5：冤枉被当反例的人

TBWA"老板要的"那句本身就是TBWA做对的事（松弛自嘲的小红书语气），拿来当"4A不会做"的证据逻辑是反的。评审要看清：**被举的反例是不是真的反例**。

## 评审输出格式

写给用户的评审回复，遵守：

1. **开头承认盲点**（避免"成型陷阱"）——"我自己的盲点是XX"
2. **按严重度排序**——🔴严重 → 🟡中等 → 🟠轻量
3. **每个问题的结构**：
   - 描述问题（1-2句）
   - **改进**：给具体重写的那句话/那个动作
4. **综合重判表格**——用打分表快速展示改进空间
5. **结尾给用户选项**——让他选方向，不代他决定

**禁忌**：
- 不说"整体来说还不错"这种安慰话
- 不把所有问题并列（没有优先级=没有价值）
- 不只描述问题不给方案
- 不在没战略判断的情况下直接写修改稿

## 迭代节奏

脚本会经历多轮。每一版保留历史：

```
v1 → 基础版
v2 → 加历史弧线和时代判断
v3 → 从"外面骂"改为"里面人叹气"
v4 → 爱德曼+蓝标具体履历入正文
v5 → 修复开场慢/动作不安全/缺自嘲/信任破绽/口号论证
```

每次新版在文件顶部加"vN 相对 vN-1 改了什么"的表格。便于用户追溯演进。

**每次迭代，都要完整重跑Pass 1-8**。不要假设"这次只改一处所以其他地方没事"——每一次改动都可能破坏其他pass的结论。

---

## 最重要的一句

**评审不是找问题，是帮AZ不浪费每一条内容。**

AZ号现在156粉，每条都是地基。一条含糊的稿子 > 一条错位的稿子。宁可告诉他"这条先不发"，也不要帮他优化一条把账号带偏的内容。

锋利 > 礼貌。诚实 > 讨好。
