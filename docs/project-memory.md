# 不现实商店官网项目记忆

## 1. 项目目标

为 `不现实商店 / Oddity Store` 搭建一个面向海外用户的官方网站。

网站要同时做到两件事：

- 介绍品牌
- 展示并售卖产品

官网不是淘宝搬运页，也不是只有品牌故事的展示页。  
当前确认的方向是：

`以产品为核心的品牌官网`

也就是说：

- 首页首先卖产品
- 品牌感和世界观要强
- 网站最终要能承载全部商品

## 2. 目标用户

核心用户：

- 机械键盘玩家
- 收藏型用户
- 喜欢主题化桌搭的人
- 海外设计向消费用户

他们更在意：

- 世界观
- 产品图像和质感
- 系列感
- 是否值得收藏

## 3. 品牌与文案方向

### 当前品牌语气

不要再使用通用的：

- `绝密`
- `classified`
- `秘密档案`

这类词不能再作为品牌自己的栏目命名核心。

品牌文案应该回到 `不现实商店` 自己的语气。

### 已确认的文案基调

来源于品牌手册与已确认表达：

- `认真不现实`
- `把那些被轻易否定的想法，认认真真地做一次`
- `每一颗键帽，都是一个平行世界的微型通道`
- `我们不为“打字”设计，而是为“叙事”与“沉浸”而造`

### 标题策略

标题要：

- 更短
- 更直接
- 不要太像说明文

当前已经统一收短，避免过长的句子型标题。

## 4. 语言策略

### 网站页面

网站不能中英混排。

必须拆成两个版本：

- 中文版
- 英文版

### 当前版本文件

- 中文版首页：[oddity-homepage-v2.html](/E:/codex/oddity-homepage-v2.html)
- 英文版首页：[oddity-homepage-v2-en.html](/E:/codex/oddity-homepage-v2-en.html)
- 英文打字机版首页：[oddity-homepage-v2-en-typewriter.html](/E:/codex/oddity-homepage-v2-en-typewriter.html)

### 语言切换

头部要有语言切换按钮：

- 桌面端放在右上工具区
- 手机端放在隐藏菜单里

### 沟通规则

在后续和用户沟通时，如果我提到英文术语，需要顺手补中文解释。

## 5. 导航结构

### 一级导航

当前确认：

- `首页`
- `键帽`
- `键盘`
- `桌垫`
- `杯子`
- `壳类`
- `关于`

右侧功能：

- `搜索`
- `登录 / 账号`
- `购物车`

### 导航交互

导航不是只跳品类页。

用户要求：

`鼠标触发品类后，可以直接弹出对应系列`

当前已经实现：

- 桌面端：悬停一级品类，弹出对应系列/分组
- 手机端：汉堡菜单内，品类可以继续展开二级子项

### 目前导航下拉分组

#### 键帽

- 全部键帽
- 魔法系列
- 大文豪系列
- 世界入口

#### 键盘

- 全部键盘
- 大文豪键盘
- 键盘保护膜

#### 桌垫

- 全部桌垫
- 文学桌垫
- 魔法桌垫
- 世界入口桌垫

#### 杯子

- 全部杯子

- 魔法塔楼杯
- 中土世界杯

#### 壳类

- 全部壳类
- 手机壳
- 耳机壳

## 6. 品类归类规则

### 键盘与键盘套件

用户已明确：

`Keyboard / 键盘` 和 `Keyboard Kit / 键盘套件` 视为同一类

### 键盘保护膜

这里说的不是键帽膜，而是：

`笔记本键盘保护膜`

因此它归到：

- `键盘`

不归到：

- `键帽`
- `壳类`

### 壳类

归到：

- 手机壳
- 耳机壳

### 杯子

必须独立一级分类。

### 大文豪键盘

必须在网站结构里明确出现，不能遗漏。

## 7. 首页结构

当前首页顺序已经确定为：

1. `Hero Carousel / 首屏轮播`
2. `Featured Now / 当前推荐`
3. `Collections / 系列入口`
4. `Browse by Category / 按品类浏览`
5. `Users and Creators / 用户与博主`
6. `What We Do / 我们在做什么`
7. `Footer / 页脚`

### 已确认的首页核心原则

- 首屏必须先卖产品
- 品牌介绍不能压过产品
- 评价、博主、自媒体、服务都要有
- 但必须按转化顺序排，不许抢首屏

## 8. 首页模块当前状态

### 首屏 Hero

已确认：

- 桌面端首屏是大图轮播
- 轮播交互参考 [Qwertykeys](https://www.qwertykeys.com/)
- 当前有滚动显现、分层进入、hover 联动等交互

后续重要规则：

- 用户会再提供新的 Hero 横幅图
- 现在的 Hero 图只是临时图

### 当前推荐

作用：

- 放当前最重要的产品
- 先让网站有商业感

### 系列入口

这块很重要，当前最新决定：

- 这块是 `系列`，不是 `品类`
- 现在是横向滑动轨道
- 带左右切换按钮
- 按钮放在下面，不放在标题旁边
- 这块的图片必须用 `方图`

当前系列卡包含：

- 魔法系列
- 大文豪系列
- 大文豪键盘
- 桌面延展
- 世界入口
- 杯欢世界

### 按品类浏览

这块和 `系列入口` 不一样。

当前最新决定：

- `品类` 这块恢复成纯文字卡
- 不带图片

也就是说目前是：

- `系列入口`：方图卡
- `按品类浏览`：文字卡

### 社区内容

首页需要保留：

- 用户评价
- 博主内容
- 工作室记录

### 我们在做什么

首页要保留服务/价值说明模块，用来建立信任。

## 9. 图片与比例规则

这是当前非常重要的硬规则。

### Hero 图

- 桌面端可以按 `21:9` 方向去做
- 但手机端不能硬套 `21:9`
- 手机端需要更适合阅读和操作的比例
- 用户后续会提供新的 Hero 图

### 1:1 规则

只要是 `1:1 / 方图` 的位置：

- 必须使用方图

### 键帽图规则

对于键帽相关的方图位：

- 优先使用 `封面图正方`
- 优先取每个系列里的首图

也就是：

- `封面图正方 / 01`

### 3:4 规则

媒体、博主、社区类图可以用：

- `3:4`

### 当前已执行的区块

- `系列入口`：方图
- `键帽相关方图位`：优先换成 `封面图正方 / 首图`

## 10. 视觉方向

### 已确认的视觉气质

整体方向仍然是：

- 档案感
- 纸张感
- 编辑感
- 收藏感

但不是“装神秘”的视觉。

### 已确认的具体规则

- 整体尽量走直角设计
- 少用“大框套小框”
- 顶部导航不要按钮框感
- 重点黄色统一用：`#fff100`

### 字体方向

英文版另外做了一个实验版：

- [oddity-homepage-v2-en-typewriter.html](/E:/codex/oddity-homepage-v2-en-typewriter.html)

它的方向是：

- 更像旧档案
- 更像打字机机打稿
- 复古英文排版感更强
